# ☸️ Kubernetes (K8s) Comprehensive Guide

This repository contains a complete overview of Kubernetes, from its origins and key features to its architectural components and deployment methods.

---

## 📖 1. What is Kubernetes?

* **Definition:** An open-source platform designed to automate the deployment, scaling, and management of containerized applications.
* **Origin:** Originally developed by **Google** (based on their internal system called *Borg*). It was open-sourced in 2014 and is now maintained by the **Cloud Native Computing Foundation (CNCF)**.
* **The Standard:** It has become the de facto standard for container orchestration, used by almost all modern tech companies.

---

## 🛠️ 2. Key Features

### Step 1: Orchestration & Resource Management

Kubernetes acts as an abstraction layer over your hardware. It decides where a container should run based on available CPU and memory, ensuring efficient resource utilization.

### Step 2: Automation & Scaling

Using objects like **Deployments**, K8s handles automatic rollouts. The **Horizontal Pod Autoscaler (HPA)** automatically adjusts the number of pods based on real-time demand.

### Step 3: Self-Healing

Kubernetes constantly monitors health. If a container crashes, it restarts it. If a Node fails, it reschedules those workloads onto healthy nodes.

### Step 4: Networking & Service Discovery

* **ClusterIP:** For internal communication.
* **LoadBalancer/Ingress:** To expose apps to the internet.
* **Service Discovery:** Pods find each other automatically using labels.

### Step 5: Declarative Configuration

You define the **Desired State** (e.g., "I want 3 replicas"). Kubernetes runs a **Reconciliation Loop** to ensure the **Actual State** always matches your definition.

### Step 6: Storage Management

Supports both stateless and stateful apps. It can attach/detach volumes (like AWS EBS or Network FS) to ensure data persists even if a pod moves to a different node.

## 🏗️ 3. Architecture & Components

![Image](https://github.com/user-attachments/assets/9643419f-cb37-46ac-a3c0-3459efb5b338)

### 🕹️ Control Plane (Master Node)

1. **API Server:** The "front door." All communication happens here.
2. **Scheduler:** Decides which node a new pod should live on.
3. **Controller Manager:** Handles node, replication, and endpoint controllers to maintain the desired state.
4. **etcd:** The cluster's "brain." A key-value store containing all cluster data.

### 👷 Worker Node

1. **Kubelet:** The agent that ensures containers are running in a pod.
2. **Kube-proxy:** Manages network rules and load balancing for the pods.
3. **Container Runtime:** The software that runs the containers (e.g., `containerd`).

## ❓ 5. Common Questions (FAQ)

**1. Is Docker mandatory for Kubernetes?**
No. Kubernetes supports multiple runtimes via the Container Runtime Interface (CRI), such as `containerd` and `CRI-O`.

**2. How is K8s different from Docker?**
Docker builds and runs individual containers. Kubernetes **orchestrates** hundreds of those containers across a cluster of many machines.

**3. What is a Microservice?**
An architectural pattern where an application is broken into small, independent services that communicate via APIs. K8s is the perfect home for this architecture.

**4. Are there alternatives?**
Yes, such as Docker Swarm, HashiCorp Nomad, and OpenShift (which is built on top of K8s).

---

## 💡 Summary: Why use Kubernetes?

It solves the "Production Problem." It ensures your application is highly available, scales with your users, and survives hardware failures without manual intervention.
