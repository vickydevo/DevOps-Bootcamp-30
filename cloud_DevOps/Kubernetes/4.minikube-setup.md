# Minikube Setup Guide (Linux)

## 1. System Requirements & Prerequisites
Before starting, ensure your system meets these minimum specs:
* **CPU:** 2 Cores or more.
* **RAM:** 2GB (4GB recommended).
* **Storage:** 20GB of free disk space.
* **Permissions:** `sudo` privileges.

---

## 2. Install and Configure Docker
Minikube needs a "driver" to run the Kubernetes cluster. Docker is the most common choice.

### Step 1: Install Docker
```bash
sudo apt update && sudo apt install docker.io -y
```

### Step 2: Configure Permissions
To run Docker without `sudo` (which Minikube prefers), add your user to the docker group:
```bash
sudo usermod -aG docker $USER && newgrp docker
```

### Step 3: Verify Docker Service
```bash
sudo systemctl start docker
sudo systemctl enable docker
docker --version
```

---

## 3. Install Minikube
Now, download the Minikube binary and move it to your executable path.

```bash
curl -LO https://storage.googleapis.com/minikube/releases/latest/minikube-linux-amd64
sudo install minikube-linux-amd64 /usr/local/bin/minikube

# Clean up the downloaded file
rm minikube-linux-amd64
```

---

## 4. Install `kubectl`
`kubectl` is the command-line tool used to communicate with the Kubernetes cluster.

### Step 1: Download and Move
```bash
curl -LO "https://dl.k8s.io/release/$(curl -L -s https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl"
chmod +x kubectl
mkdir -p ~/.local/bin
mv ./kubectl ~/.local/bin/kubectl
```

### Step 2: Update PATH
Ensure your system knows where to find `kubectl`:
```bash
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc
```

### Step 3: Verify `kubectl`
```bash
kubectl version --client
```

---

## 5. Start and Verify Minikube
With everything installed, it's time to spin up your local cluster.

### Step 1: Start Minikube
We explicitly tell Minikube to use the Docker driver.
```bash
minikube start --driver=docker
```
**![Image](https://github.com/user-attachments/assets/f513eea3-79dd-49d3-beca-1c25e23fe1d9)**

### Step 2: Check Cluster Status
```bash
minikube status
```
***![Image](https://github.com/user-attachments/assets/29cb950f-b7e7-4e87-8e95-2e384d1b2c62)***

### Step 3: Test Cluster Access
Run this command to see the internal nodes of your new cluster:
```bash
kubectl get nodes
```

> **Pro-Tip:** If you ever run into driver issues, you can reset everything with `minikube delete` and try starting again.``

# Minikube Setup & Verification Guide

## 6\. Verify Cluster Workloads

Once the cluster is running, use these commands to ensure everything is initialized correctly.

### Step 1: List All Managed Workloads

This provides a high-level overview of Pods, Services, Deployments, and ReplicaSets.

```bash
kubectl get all
```

### Step 2: Detailed Workload Inspection

To explicitly list all workload types with detailed information:

```bash
kubectl get deployments,replicasets,jobs,cronjobs,pods,statefulsets,daemonsets
kubectl get all --show-kind --output=wide
```

-----

## 7\. Access the Kubernetes Dashboard

The dashboard provides a powerful web-based UI for managing your cluster.

### Step 1: Start the Dashboard

Enable the dashboard add-on within Minikube:

```bash
minikube dashboard
```

### Step 2: Set Up External Proxy (For Remote/EC2 Access)

If you are accessing this from outside the local machine (like an EC2 instance), run this in a second terminal window to allow traffic:

```bash
kubectl proxy --address='0.0.0.0' --accept-hosts='^*$'
```

## 8\. Maintenance & Cleanup

Commands to manage the lifecycle of your Minikube instance.

### Stop the Cluster

Pauses the nodes but retains all configurations and deployments:

```bash
minikube stop
```

### Delete the Cluster

Completely removes the cluster and frees up all system resources (RAM/Storage):

```bash
minikube delete
```

### Purge All Data

If you want a completely fresh start including removing cached images:

```bash
minikube delete --all --purge
```
